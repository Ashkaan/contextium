Run the **Think** move of the Contextium Loop.

1. Understand what is actually being asked. If it is a real decision (architecture, a vendor, anything hard to reverse), present the options and trade-offs and wait. If it is execution, plan it and move.
2. Write a lean SPEC with four sections: what was asked, what success looks like (including the 0 / 1 / empty / max / error edges), the files to touch, and the exact commands that prove it works.
3. Present the SPEC and get explicit sign-off before building.

Stop here. The Do move happens in a fresh chat so a context that grew attached to the plan is not the one that judges the build.
