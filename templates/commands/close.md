Run the **Wrap** move of the Contextium Loop.

1. Write today's journal entry at `journal/YYYY-MM-DD.md` using the labeled markers: Action / Changes / Decisions / Issues / Lessons / Next. One file per day; append if it already exists.
2. Update any docs the change drifted (README, SPEC, project notes).
3. Commit with a verb-led subject (`add` / `fix` / `update` / `refactor` / `remove` …) that reads as memory in the log. No AI co-author trailers.

The git log records WHAT changed; the journal records WHY. Reconstructing a past decision needs both, so write both.
